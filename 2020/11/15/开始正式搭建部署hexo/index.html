开始正式搭建部署hexo

发表于 2020-11-15 |
字数统计: 723 字 | 阅读时长 ≈ 3 分钟

原文链接：https://zhuanlan.zhihu.com/p/295428650

1、用管理员的身份打开CMD然后输入
npm install -g cnpm –registry=https://registry.npm.taobao.org
（因为国内npm镜像源很慢，所以这里利用npm来安装一个淘宝的镜像源cnpm）
点击回车进行安装

2、输入cnpm install -g hexo-cli进行全局安装

3、安装完成后，进入C盘根目录（此处盘符不限于C盘），输入hexo init blog来建立一个名字叫blog的文件夹并初始化为博客，需要一点时间。

4、初始化后，blog文件夹下多出一下文件

5、使用“C:\blog”命令进入blog文件夹，输入hexo s或者hexo server在本地启动本地博客预览，在浏览器中输入http://localhost:4000 来访问 ，或点击下方链接访问

预览后效果图

利用Ctrl+C断开

6、利用hexo n来创建一片文章名字叫“我的第一篇博客文章”

7、进入_posts文件夹下使用dir ./b命令进行查看
输入命令echo 123456789 >>我的第一篇博客文章.md来为文章添加123456789

8、返回blog文件夹，输入hexo g生成静态页面，然后输入hexo s 进行查看

预览后效果图

9、在网页中输入 http://github.com 登录你的github账号，进入下面界面，输入邮箱和密码进行登录

10、点击Create repository创建一个新的仓库

或者按照下图操作也可以


继续安装下图操作

创建完成

11、打开_config.yml文件进行配置



12、输入npm install –save hexo-deployer-git


13、输入邮箱和名字（输入你的注册GitHub的邮箱与名字）
git config –global user.email “你的邮箱
git config –global user.name “你的名字

14、输入hexo d部署到GitHub，出现以下窗口，点击在网页进行授权，cmd窗口稍加等待

15、完成后登陆github刷新仓库发现多出以下文件，此时就完成了部署

在浏览器中输入你的用户名+http://github.io即可访问

出错解决方法

输入set-ExecutionPolicy RemoteSigned
输入A

所需命令如下

npm install -g cnpm –registry=https://registry.npm.taobao.org
cnpm install -g hexo-cli
cd
hexo init blog
cd “C:\blog”
dir ./b
hexo s
hexo n “我的第一篇博客文章”
cd “C:\blog\source_posts”
dir ./b
echo 123456789 >>我的第一篇博客文章.md
hexo g
hexo s
cd “C:\blog”

_config.yml配置如下:
deploy:
type: git
repo:
branch: master

npm install –save hexo-deployer-git
git config –global user.email “邮箱
git config –global user.name “用户名
hexo d

命令解释

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面
hexo server 开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy 部署到GitHub
hexo help 查看帮助
hexo version 查看Hexo的版本
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
hexo -v == hexo version
